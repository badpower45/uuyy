# Offline-First Mobile App

## Overview

التطبيق الموبايل يدعم وضع **Offline-First** كامل — يحفظ كل البيانات محلياً على الجهاز ويعمل بدون إنترنت! 📱

## Features

### ✅ الحفظ المحلي (AsyncStorage)

كل هذه البيانات تُحفظ تلقائياً على جهاز المستخدم:

- **بيانات المستخدم** — الاسم، الهاتف، الرصيد، الرتبة، إحصائيات
- **الموقع الحالي** — أحدث موقع GPS مع الدقة والاتجاه
- **الأرباح الأسبوعية** — كل يوم من الأسبوع الماضي مع التفاصيل
- **الطلبات** — الطلبات النشطة والمتاحة
- **العمليات المعلقة** — تحديثات الموقع والأرباح المراد تزامنها

### 📍 Offline-First Strategy

عندما يكون التطبيق **بدون إنترنت**:

1. **عرض البيانات المحفوظة** ← تظهر فوراً من الذاكرة المحلية ❌ بلا تأخير
2. **حفظ العمليات الجديدة** ← كل طلب أو موقع يُحفظ محلياً في قائمة انتظار
3. **إظهار حالة الاتصال** ← يظهر للمستخدم أن التطبيق أوفلاين

### 🔄 التزامن التلقائي (Auto-Sync)

عندما يعود الاتصال:

1. **التزامن التلقائي** ← يبدأ تلقائياً بدون أن يفعل المستخدم شيء
2. **أولويات الإرسال** — 
   - أولاً: تحديثات الموقع (الأهم للخرائط)
   - ثانياً: تحديثات الأرباح والطلبات
3. **إعادة محاولة ذكية** ← إذا فشل شيء، يحاول مرة أخرى 3 مرات مع زيادة التأخير
4. **تحديث البيانات** ← بعد التزامن الناجح، يسحب البيانات المحدثة من السيرفر

## Architecture

```
┌─────────────────────────────────────────┐
│      AppContext (State Management)      │
│  مدير الحالة المركزي للتطبيق           │
└────────────┬────────────────────────────┘
             │
      ┌──────┴──────┬────────────┬─────────┐
      │             │            │         │
      ▼             ▼            ▼         ▼
   Login      Location      Earnings    Orders
   التسجيل      الموقع       الأرباح    الطلبات
      │             │            │         │
      └──────┬──────┴────────────┴─────────┘
             │
      ┌──────▼──────────────────────┐
      │   OfflineStorage            │
      │ (AsyncStorage Manager)      │
      │ مدير التخزين المحلي         │
      └──────┬──────────────────────┘
             │
      ┌──────▼──────────────────────┐
      │   Device Storage            │
      │ (AsyncStorage/SQLite)       │
      │ التخزين على الجهاز         │
      └─────────────────────────────┘

      ┌────────────────────────────┐
      │   SyncManager              │
      │ (Background Sync)          │
      │ مدير التزامن               │
      └────────────┬───────────────┘
             │
      ┌──────▼──────────────────────┐
      │   API Client               │
      │ (Network Requests)         │
      │ عمليات الشبكة              │
      └──────┬──────────────────────┘
             │
      ┌──────▼──────────────────────┐
      │   Backend API              │
      │ (Supabase PostgreSQL)      │
      │ السيرفر                    │
      └─────────────────────────────┘
```

## Code Files

### 1. `lib/offlineStorage.ts`
مدير التخزين المحلي — يحفظ ويسترجع كل البيانات

**الدوال الرئيسية:**
- `saveAuth()` — حفظ بيانات التسجيل
- `getAuth()` — استرجاع بيانات التسجيل
- `saveLocation()` — حفظ الموقع الحالي
- `getLocation()` — استرجاع الموقع المحفوظ
- `saveWeeklyEarnings()` — حفظ الأرباح
- `getWeeklyEarnings()` — استرجاع الأرباح
- `addPendingLocation()` — إضافة موقع للقائمة المعلقة
- `getPendingLocations()` — الحصول على تحديثات الموقع المعلقة
- `addPendingEarning()` — إضافة ربح للقائمة المعلقة
- `getPendingEarnings()` — الحصول على الأرباح المعلقة

### 2. `lib/syncManager.ts`
مدير التزامن — يزامن العمليات المعلقة عندما يعود الإنترنت

**الدوال الرئيسية:**
- `startAutoSync()` — بدء التزامن التلقائي (يُستدعى عند التسجيل)
- `stopAutoSync()` — إيقاف التزامن التلقائي (يُستدعى عند الخروج)
- `syncPendingData()` — مزامنة فورية
- `getSyncStats()` — الحصول على إحصائيات التزامن
- `hasPendingOperations()` — التحقق من وجود عمليات معلقة

### 3. `context/AppContext.tsx`
مدير الحالة الرئيسي — يدمج الـ offline storage مع الـ API

**التغييرات:**
- `login()` — يحفظ البيانات في OfflineStorage ويبدأ التزامن
- `logout()` — يحذف البيانات المحفوظة ويوقف التزامن
- `refreshEarnings()` — يحفظ الأرباح محلياً عند النجاح، يسترجع من التخزين عند الفشل
- `applyDriverLocation()` — يحفظ كل تحديث موقع محلياً
- `pushLocationToBackend()` — إذا فشل الإرسال، يقف العملية في قائمة الانتظار
- `seedMockWalletData()` — يقف البيانات الوهمية إذا كان أوفلاين

## User Experience

### السيناريو 1: استخدام عادي (أونلاين)
```
1. المستخدم يسجل الدخول
   ↓ يحفظ بيانات التسجيل محلياً
   ↓ يبدأ التزامن التلقائي
   
2. ينتقل لموقع ما أثناء التوصيل
   ↓ كل 5 ثواني: يُحفظ الموقع محلياً
   ↓ يُرسل الموقع للسيرفر
   ↓ الخريطة تظهر الموقع الحي
   
3. يستقبل طلب جديد
   ↓ يُحفظ محلياً
   ↓ يُرسل التقبل للسيرفر
   ↓ الأرباح تُحدّث تلقائياً

4. يسجل الخروج
   ↓ جميع البيانات تُحذف محلياً
```

### السيناريو 2: فقدان الإنترنت (أوفلاين)
```
1. أثناء التوصيل، ينقطع الإنترنت
   ↓ التطبيق يستمر بدون توقف ✓
   ↓ كل تحديثات الموقع تُحفظ محلياً
   ↓ تظهر رسالة "أوفلاين"
   
2. يصل للعميل ويكمل الطلب
   ↓ يضغط "تم التسليم"
   ↓ البيانات تُحفظ محلياً
   ↓ الأرباح تُحدّث محلياً
   
3. يعود الإنترنت
   ↓ التزامن يبدأ تلقائياً
   ↓ جميع الأموقع والأرباح تُرسل
   ↓ البيانات تُحدّث من السيرفر
   ↓ رسالة "متصل" تظهر
```

## Storage Capacity

- **AsyncStorage** قادر على تخزين عشرات الآلاف من بيانات الموقع والطلبات بسهولة
- لكل موقع GPS: ~200 bytes
- لكل ربح: ~150 bytes
- إذا كان 100 موقع في الساعة لمدة 8 ساعات = 800 موقع = ~160 KB فقط!

## Sync Error Handling

إذا فشل التزامن:

```
Retry 1: بعد 2 ثانية
Retry 2: بعد 4 ثواني  
Retry 3: بعد 8 ثواني
After 3 retries: يُحفظ في قائمة خاصة للعمليات الفاشلة
```

المستخدم يرى:
- ⏳ "التزامن قيد التقدم..."
- ❌ "فشل التزامن، إعادة محاولة..."
- ✓ "تم التزامن بنجاح"

## Best Practices

1. **استدعي `startAutoSync()` عند التسجيل** ✓ (في AppContext)
2. **استدعي `stopAutoSync()` عند الخروج** ✓ (في AppContext)
3. **احفظ البيانات الجديدة محلياً فوراً** ✓ (في refreshEarnings و applyDriverLocation)
4. **في حالة الفشل، جرب البيانات المحفوظة** ✓ (في refreshEarnings)
5. **أرسل الأخطاء للـ sync queue** ✓ (في pushLocationToBackend و seedMockWalletData)

## Summary

التطبيق الآن:
- 🔒 يحفظ كل بيانات المستخدم محلياً
- 📍 يحفظ تحديثات الموقع حتى بدون إنترنت
- 💰 يحفظ الأرباح والطلبات محلياً
- 🔄 يزامن تلقائياً عند العودة للإنترنت
- ❌ يعمل بدون إنترنت — لا توقف الخدمة!

**يمكن للسائق الآن:**
- ✅ يسجل دخول مرة واحدة ولا يحتاج يسجل مرة أخرى
- ✅ يوصل الطلبات حتى بدون إنترنت
- ✅ يرى موقعه والأرباح والطلبات في أي وقت
- ✅ جميع البيانات تتزامن تلقائياً عند العودة للإنترنت
